# N30+ Candidate Directions

This companion file expands the compact candidate table in
[N30+ Experiment Catalog Roadmap](N30_plus_experiment_catalog_roadmap.md).

It is not a fixed schedule. Candidate names below are orientation labels for
future experiment design. A concrete experiment may split, merge, postpone, or
revise these directions if it preserves source-basis discipline, debt records,
and claim boundaries.

Definitions for primitives, building blocks, motifs, regimes, debt, failure,
composition, and claim ceilings are in the main
[N30+ Experiment Catalog Roadmap](N30_plus_experiment_catalog_roadmap.md).
This companion only expands candidate directions; it is not a substitute for
source artifacts or roadmap claim-boundary rules.

All candidates inherit:

- the roadmap's source-basis rule;
- AP4/AP5 NAT4 carry-forward;
- the debt and failure taxonomies;
- the composition rules;
- the claim-boundary policy.

## Naming convention

The names below use three kinds of orientation labels:

- **N-numbered candidates** name relatively concrete experiment proposals.
- **Track labels** name thematic families that may produce more than one
  experiment, especially when discovery work is still broad.
- **Later probes** name long-horizon targets that should wait for admitted
  primitives, building blocks, or motifs.

The labels are planning handles, not a required schedule. If a future
experiment splits a candidate, the split should preserve the same evidence
boundaries and should state which part is contract/admission work and which
part is positive runtime evidence.

## Candidate relationship map

The candidates are related, but not strictly linear:

```text
primitive track + Candidate N30
  -> candidate N31 / N32 / N33 / N34 building-block probes
  -> candidate N35+ motif experiments
  -> later regime probes
```

Candidate N30 and the primitive track are natural entry points because later
building-block probes need admitted or at least explicitly bounded primitive
ingredients. N31-N34 can run in parallel when their required primitives and
source artifacts exist. N35+ motif experiments should normally consume at least
two controlled building-block candidates. Later regime probes should wait for
motif-level evidence and should keep the roadmap's regime claim boundary
visible.

## Candidate N30: Shared-Medium Catalog Grammar and First Admissibility Probe

Primary layer:

```text
catalog_contract with primitive and building-block probe
```

Purpose:

Define the first N30+ catalog artifacts and test one small shared-medium
building-block candidate.

When instantiated as a concrete experiment, this candidate may be split into:

```text
N30-A: catalog contract and admission schema
N30-B: first shared-medium primitive/building-block positive probe
```

N30-A would define artifact schemas, admission gates, debt taxonomy, claim
ceilings, and source-basis requirements.

N30-B would test one small shared-medium candidate without treating the roadmap
itself as evidence.

This split is a general optional pattern for N30+ work: when a candidate is too
dense, separate the contract/admission pass from the positive runtime probe.
The split should not let contract language count as positive evidence.

Possible target:

```text
medium trace + support gradient -> shared-medium support redistribution
```

Questions:

- Can a medium trace be observed as a primitive?
- Can a support gradient be observed as a primitive?
- Can the two compose into a source-current redistribution dynamic?
- Does redistribution change basin persistence, capacity, or eligibility?
- Are direct message/reward/producer routes blocked?
- What producer and medium debt remains?

Claim ceiling:

```text
shared-medium support-redistribution building-block candidate
```

Blocked relabels:

```text
communication, cooperation, helping, agency, ecology
```

## Candidate Primitive Track: Ecology-Specific Primitive Discovery

Primary layer:

```text
primitive
```

Purpose:

Search for primitives that become necessary only when shared-medium ecology is
attempted.

This track is broad by design. It is the downward-discovery lane from the main
roadmap: if a building block or motif fails because a substrate distinction is
missing, the primitive track is where that missing distinction can become its
own controlled probe.

Possible primitives:

- medium trace;
- support gradient;
- boundary permeability;
- latent basin;
- ghost aftereffect;
- parent-basin modulation;
- resource-capacity coupling;
- fragmentation threshold;
- medium debt marker.

Claim ceiling:

```text
ecology-specific primitive candidates
```

## Candidate Building-Block Track: First Building-Block Contract

Primary layer:

```text
building_block
```

Purpose:

Create and validate the first `building_block_contract.v1` artifact.

This candidate matters because later ecology work needs reusable contracts,
not just prose descriptions. A building-block contract should make a candidate
block consumable by future experiments without letting those experiments
silently inherit stronger claims than the block established.

The contract validation pass should check whether the artifact shape can record:

- consumed primitives and their source artifacts;
- structural condition and dynamics generator;
- carrier surface and source-current observables;
- positive signature and negative controls;
- composition inputs and outputs;
- producer residue, medium debt, naturalization debt, and transfer scope;
- claim ceiling and blocked relabels.

It would not prove the target block by itself. It would only show that the
contract can carry the evidence boundary needed for later positive probes.

Suggested fields:

```text
block_id
required_primitives
structural_condition
dynamics_generator
carrier_surface
source_current_observables
positive_signature
negative_controls
composition_inputs
composition_outputs
producer_residue
medium_debt
naturalization_debt
transfer_scope
claim_ceiling
blocked_relabels
```

Possible target:

```text
shared-medium support redistribution
```

## Direction Addendum: N30 Selected Starting Point

The candidate list above is retained as the broader N30+ orientation. The first
concrete N30 experiment is intentionally narrower than the original
support-redistribution candidate.

Selected N30:

```text
N30 - Minimal Shared-Medium Participation
primary layer = primitive
secondary observation = trace-mediated eligibility primitive / building-block candidate
```

Reason for the refinement:

```text
shared medium is not a loose synonym for a shared graph, channel, message bus,
or carrier surface. It is a defined relation grammar from the shared-medium
essay and engineering spec.
```

N30 therefore starts by grounding the smallest admissible relation form:

```text
participant carrier
  perturbs declared medium surface
trace or surface state persists / decays
later participant eligibility, cost, support, routing, susceptibility, or
capacity changes because of that surface
```

This keeps the first positive target near:

```text
M1 = boundary perturbation
M2 = trace-mediated influence / trace-mediated eligibility
```

and blocks:

```text
shared-medium coordination
parent-basin modulation
resonant alignment
native shared-medium organization
communication
cooperation
agency
ecology regime
```

Updated near-term candidate sequence:

```text
N30:
  Minimal Shared-Medium Participation
  primitive / relation-admission experiment

N31:
  Shared-Medium Support Redistribution
  building-block candidate, consuming N30 if N30 closes cleanly

N32:
  Slow-Memory Conductance Trace
  building-block candidate

N33:
  Boundary-Mediated Exchange
  building-block candidate

N34:
  Generative/Extractive Shared-Medium Relation
  building-block candidate consuming N28-style environment-effect discipline
```

The original Candidate N30 contract/probe split remains useful as an
operational pattern inside N30 or N31:

```text
contract/admission pass first
positive runtime probe second
```

but contract language must not count as positive shared-medium evidence.

## Direction Addendum: Cross-Project Spiral

The candidate sequence above should not be read as a closed LGRC-only backlog.
After N30 grounds minimal shared-medium participation, the next candidate choice
should be allowed to depend on a demand/composition pass in the agentic ecology
project.

Recommended operating rhythm:

```text
N30 in graph-reflexive-coherence:
  minimal shared-medium participation gate

agentic ecology demand pass:
  identify which shared-medium primitives and building blocks ecology actually
  needs, without treating those demands as evidence

N31+ in graph-reflexive-coherence:
  test one demanded primitive or building block with source-current artifacts,
  replay, controls, debt accounting, and claim ceilings

ecology motif pass:
  compose admitted or bounded candidates into motif-level probes when enough
  building blocks exist
```

This means N31 is not automatically fixed by the original candidate order.
Shared-medium support redistribution remains a strong candidate, but the
agentic ecology demand pass may show that another building block should come
first, such as:

```text
slow trace
boundary permeability
latent basin / ghost aftereffect
resource-capacity coupling
role/cargo-shaped susceptibility
fragmentation threshold
```

The constraint is that demand is not proof. Agentic ecology may expose the need;
LGRC must still test whether the needed distinction is source-current,
replayable, control-clean, and claim-bounded.

This keeps the catalog bidirectional:

```text
upward:
  primitive -> building block -> motif -> regime

downward:
  missing ecology behavior -> missing block -> missing primitive -> LGRC probe
```
