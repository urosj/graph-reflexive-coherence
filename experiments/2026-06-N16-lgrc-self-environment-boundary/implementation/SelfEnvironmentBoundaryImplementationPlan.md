# N16 Self / Environment Boundary Implementation Plan

## Purpose

N16 tests whether internal support-relevant state and external resource or
perturbation state can be separated in artifacts and controls.

N16 should be a requirements-discovery experiment for stable basin boundaries
across lineage-derived boundary-state and operational challenge-class
interaction cells, not a complete self/environment taxonomy.

The guiding question is:

```text
What does an LGRC basin need in order to remain a basin when it is exposed to
boundary challenges?
```

N16 targets `AP6` at most:

```text
AP6 = self/environment boundary candidate
```

The expected final ceiling, if supported, is:

```text
artifact_level_ap6_self_environment_boundary_candidate
```

This is not selfhood, personhood, identity acceptance, semantic goal ownership,
intention, semantic choice, agency, native support, fully native integration,
or unrestricted agency.

N16 explicitly excludes:

```text
selective uptake
resource assimilation
porous boundary function
agency-like boundary behavior
semantic recognition
organism or life claims
full failure taxonomy
```

## Evidence Strategy

Iteration 1 should first check whether historic artifacts directly support
internal/external state separability. If direct support exists, use it only as
source-backed, claim-clean evidence and still run the N16 controls.

The strongest expected proof path is a constructed boundary candidate built
from the old best closed claims:

```text
N15 AP5 -> endogenous target/proxy formation axis
N14 AP4 -> consequence-sensitive selection axis
N13 AP3 -> support-seeking regulation axis
N08 -> memory/context axis
N09 -> bounded-regulation axis
N12 NAT4 -> readiness-only context
```

The construction must preserve each source experiment's claim boundary. It may
combine old best claims to form an AP6 candidate, but it must not promote N15
into goal ownership, N14 into intention, N13 into selfhood, or N12 into native
support.

The implementation target is a bounded two-axis matrix:

```text
lineage-derived boundary state
x
operational challenge class
```

N16 should freeze both axes, then run main-effect sweeps plus selected
interaction probes. It should report which requirements were satisfied or
failed, not only whether a boundary was detected.

## Pre-Implementation Contract

### Terminology

Use `internal support-relevant state` for artifact-visible state that belongs
to the support, regulation, memory, or target/proxy side of the system state
being maintained.

Use `external resource state` for artifact-visible state representing resource,
route, substrate, or outside conditions beyond the internal support surface.

Use `external perturbation state` for artifact-visible perturbations that act
on internal support or resource conditions from outside the internal support
surface.

Use `external structured state` for coherent outside structure that creates
false-positive boundary pressure without necessarily disrupting the candidate
basin.

Use `boundary-crossing trace` for a field-level record showing how a scheduled
response, route, target/proxy condition, or perturbation crosses between the
internal and external sides. This is not an action-perception loop claim.

Use `basin` for an artifact-visible coherent region whose internal support or
coordination remains distinguishable from its background across a case.

Use `background` for passive or incoherent non-basin outside state.

Use `structured external coherence` for coherent outside structure that must
not be mistaken for a basin boundary.

Use `shared medium` for a substrate containing more than one candidate basin
plus possible coupling channels.

### Boundary State Axis

The B-axis is lineage-derived. Iteration 1 must derive or reject each B-state
from prior N** source records rather than treating the labels below as
freestanding boundary theory. Each B-state must record:

```text
lineage_sources
inherited_closed_claims
constructed_support
unsupported_extension
required_N16_evidence
claim_ceiling
```

```text
B0 null / external coherence only
    Active null. Prior controls can support rejection of relabeled external
    structure, but no boundary claim is allowed.

B1 localized basin partition
    Basin-like inside/outside partition and boundary edges can be extracted.
    Main lineage: N03 basin-loop surfaces and N04 boundary-coupling taxonomy.

B2 support-persistent basin
    Boundary remains stable across time/replay. Main lineage: N07
    identity/support persistence plus later AP support/replay records.

B3 regulated repair / reabsorption boundary
    Boundary can reabsorb leakage or recover after disruption. Main lineage:
    N09 bounded recovery, N13 support response, and N15 drift/replay controls.

B4 coupled multi-basin separability candidate
    Separate basins remain distinct while coupled through shared substrate.
    This is the weakest inherited state: N08/N14/N15 supply partial analogs,
    but N16 must produce the separability evidence.
```

### Challenge Class Axis

The C-axis is not an inherited environment taxonomy. It is an operational
challenge-class basis selected to isolate fundamental ways a boundary can be
stressed, confused, breached, or coupled.

```text
C0 quiet reference
    Calibration condition with no intended boundary stress.

C1 unstructured perturbation
    Random or noisy disturbance around or across the boundary.

C2 directional flux
    One-sided pressure, drift, or flow across the boundary.

C3 structured external coherence
    Active null: coherent outside pattern that must not be mistaken for the
    basin boundary or self-region. This is external structured state that
    creates false-positive pressure, not perturbation by default. It becomes
    perturbation only if a crossing or disruption is explicitly recorded.

C4 breach and repair
    Local boundary disruption followed by a test for reclosure, reabsorption,
    or fail-closed classification.

C5 coupled neighbor / shared medium
    More than one candidate basin interacting through a shared substrate.
```

### Selected Interaction Cells

```text
B0 x C3
    structured external coherence active null; must reject false boundary

B1 x C2
    weak detectable boundary under flux; likely failure threshold

B2 x C1
    persistent boundary under unstructured perturbation

B3 x C4
    breach and repair / reclosure probe

B4 x C5
    multi-basin exclusivity in shared medium
```

B3 repair/reabsorption cells are unlocked only after B2 has been evaluated
under C0 quiet reference, C1 unstructured perturbation, and C2 directional
flux, or after those evaluations produce explicit blockers. Repair cannot be
used to hide an uncalibrated or unsupported persistent boundary.

B4 can appear in Iteration 5 as a directional-flux stress row, but its primary
target is B4 x C5 in Iteration 6. If the multi-basin substrate is not
sufficiently source-backed, the Iteration 5 B4 row must be `partial` or
`not_applicable` rather than promoted as separability evidence.

### AP5 To AP6 Bridge

N15 AP5 supplies source-current target/proxy formation:

```text
target/proxy condition is generated before downstream use from runtime-visible
artifact state
```

N16 AP6 requires an additional boundary crossing:

```text
internal support-relevant state and external resource/perturbation state are
separately represented, traceable, and controlled
```

AP6 is not a relabel of AP5. N16 uses AP5 as the strongest available
construction substrate, but the AP6 candidate must add internal/external state
separation with its own dependency trace, budget validity, controls, and replay.
