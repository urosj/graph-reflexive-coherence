# N30 - LGRC Minimal Shared-Medium Participation

N30 is the first N30+ experiment. It refines the earlier “shared-medium support
redistribution” candidate into a lower and cleaner first target:

```text
minimal shared-medium participation
```

N30 should not begin by claiming communication, coordination, cooperation,
agency, ecology, or native shared-medium organization. It should first ask
whether the term **shared medium** can be grounded in source-current LGRC
evidence at the smallest admissible relation form.

## Core Question

```text
Can a minimally stable participant perturb a declared medium surface, leaving a
source-current trace or surface change that later alters eligibility, cost,
support, routing, susceptibility, or capacity for a participant, without direct
sender-receiver scripting?
```

## Why N30 Exists

The N30+ roadmap initially pointed toward a catalog contract plus a first
shared-medium support-redistribution probe. That remains useful, but it is too
high-level as the first N30 target.

Before support redistribution, conductance memory, boundary exchange, or
generative/extractive shared-medium relations can be claimed, N30 must define:

```text
participant gate:
  what is stable enough to perturb or encounter a medium surface?

medium gate:
  what surface changed, how did it persist or decay, and what later eligibility
  changed because of it?

debt/control gate:
  what direct-message, global-controller, hidden-producer, label-only, or
  producer-story path was blocked or recorded as medium debt?
```

## N27 and N28 Role

N30 must not start as if “participant” and “medium” are empty abstractions.

N27 supplies the participant-continuity discipline:

```text
configuration/topology recognizability across transfer and replay,
without semantic identity or selfhood.
```

N28 supplies the environment-effect discipline:

```text
generative, extractive, competitive, and neutral environment reshaping,
without cooperation, exploitation, ecology, or agency.
```

N30 uses those as guardrails:

```text
participant continuity cannot be mere label persistence;
medium effect cannot be generic redistribution or semantic story.
```

Specific N27/N28 source artifacts are enumerated in Iteration 1. The README
names their role; it does not substitute for the source inventory.

## Local Relation Scope

N30 imports the shared-medium coordination ladder only at the lowest rungs:

```text
M0 = direct message passing, valid as boundary abstraction or scaffold
M1 = boundary perturbation
M2 = trace-mediated eligibility / trace-mediated influence
M3 = shared-field co-response, optional and not required for N30 closeout
M4 = parent-basin modulation, blocked/future
M5 = resonant alignment, blocked/future
M6 = native shared-medium organization, blocked/future
```

N30’s positive target is M1/M2. M3 may be recorded only as a secondary
candidate if evidence naturally appears.

## Participant Scope

N30 does not require stable agency or selfhood. It requires only a minimally
stable participant:

```text
participant_min =
  identifiable carrier
  + bounded persistence window
  + source-current interaction with medium
  + distinguishable contribution or response
```

Possible carrier forms include a basin, boundary region, packet loop, latent
geometry, ghost aftereffect, localized support structure, or trace structure.

For N30:

```text
P1 = attributable contributor/respondent within one event chain
P2 = same carrier remains recognizable across a bounded replay window
```

N30 needs P1/P2 for minimal shared-medium participation. It does not need a
self, agent, semantic identity, or autonomous actor.

## Coupled Relation-Chain Requirement

N30 must not pass because a participant appears somewhere, a trace appears
somewhere else, and a later difference appears somewhere else.

A positive row must link them in one relation chain:

```text
participant carrier
  -> perturbation event
  -> declared medium surface change
  -> persisted/decayed trace
  -> later eligibility/susceptibility change
  -> controls showing dependence on that medium trace
```

If those components are only separately present, the row is partial or blocked.

## Sharedness Gate

A medium surface must be non-private relative to the participant carrier.

Allowed sharedness forms include:

```text
boundary_accessible
shared_local
shared_global
route/conductance/susceptibility surface
packet/event-history surface visible to later eligibility
```

If only the original participant can encounter the change, classify the result
as a medium-mediated self-aftereffect candidate unless a separation argument and
controls justify a shared-medium reading.

## Participant / Medium Separation Audit

N30 must record whether the participant carrier and medium surface are distinct.
If they collapse into the same object, the row cannot close as minimal
shared-medium participation unless it provides a separation argument and
controls that show the medium surface is not merely private participant state.

## Claim Ceiling

The intended maximum N30 closeout language is:

```text
minimal shared-medium participation primitive
trace-mediated eligibility primitive / building-block candidate
```

N30 must not claim:

```text
communication
coordination

agency
selfhood
identity acceptance
native shared-medium organization

```

## Initial Handoff Target

If N30 succeeds, the next natural building-block experiment is not generic
ecology. It is a controlled shared-medium building block such as:

```text
N31 = shared-medium support redistribution
```

Other later candidates include slow-memory conductance trace,
boundary-mediated exchange, latent basin / ghost aftereffect, and
generative/extractive shared-medium relation.
