# N30 Minimal Shared-Medium Participation Implementation Plan

## Purpose

N30 grounds the term **shared medium** in the graph/LGRC experiment stack before
later work attempts richer shared-medium building blocks.

It does not try to prove shared-medium coordination. It tests the smallest
admissible relation form:

```text
minimally stable participant
  perturbs declared medium surface
changed medium surface leaves trace or surface state
later participant eligibility / cost / support / routing / susceptibility /
capacity changes because of that surface
```

The target claim is:

```text
minimal shared-medium participation primitive
```

with a possible secondary observation:

```text
trace-mediated eligibility primitive / building-block candidate
```

## Conceptual Direction

N30 is explicitly based on the shared-medium essay and engineering spec, but it
does not import their full ambition as evidence.

## Iteration Plan

### Iteration 1. Source Inventory And Method Admission

Build the source inventory. Admit shared-medium sources as method/boundary
sources, N27/N28 as load-bearing guardrails, and N29 as bridge context. Do not
open positive N30 evidence.

### Iteration 2. Participant / Medium Schema Freeze

Freeze participant ladder, medium relation ladder, required fields, source
artifact admissibility, controls, debt status enums, and closeout ladder.

### Iteration 3. Active Nulls And Failure Baselines

Instantiate fail-closed nulls for message-only, label-only, hidden producer,
global controller, post-hoc trace, participant drift, generic redistribution,
and semantic relabel paths. No positive evidence.

### Iteration 4. Minimal Participant Admissibility Probe

Test whether a participant carrier can be attributed and replayed across a
bounded window without selfhood, identity, or agency promotion.

### Iteration 5. Medium Surface Perturbation / Trace Probe

Test whether the participant perturbs a declared medium surface and leaves a
source-current trace or surface change with persistence/decay evidence.

### Iteration 6. Later Eligibility / Susceptibility Probe

Test whether later participant eligibility, cost, support, routing,
susceptibility, or capacity changes because of the changed medium surface.

### Iteration 7. Replay, Controls, And Medium Debt Matrix

Run artifact replay, duplicate replay, snapshot/load replay, message-removal,
medium-freeze, trace-shuffle, false-trace, decay, susceptibility-inversion,
hidden-producer, and semantic relabel controls.

### Iteration 8. Classification And N31 Handoff

Classify the strongest supported N30 result. If warranted, close at N30-C6 and
handoff to N31 as a support-redistribution or trace-mediated eligibility
building-block candidate. Do not claim shared-medium coordination or native
shared-medium organization.
