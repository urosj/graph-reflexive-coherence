# Hypothesis B: Native Geometry-Mediated Trail Memory

## Statement

Route use changes declared LGRC geometry, topology, support, packet/loop, or
other native substrate state. Future flux or routing changes because native
dynamics follow that changed substrate. The trail is a derived observable of
the substrate, not an independent `memory_strength` substance.

## Candidate Mechanisms

Possible native mechanisms include:

```text
edge split or inserted node
declared node/edge geometry change
support-shape or local coupling geometry change
topology event with lineage and state reabsorption
packet/loop residue visible in existing state
route conductance-like metric if represented as declared substrate state
```

The important distinction is that future routing must change because the
substrate changed, not because an experiment-local scalar was read as a score.

## Acceptance Requirements

A valid native geometry-mediated trail result should satisfy:

```text
no independent memory_strength field
trail observable derived from node/edge/packet/topology/support state
route use changes geometry/topology/support through declared LGRC mechanisms
future routing changes because native flux follows changed geometry
active coherence and packet budgets remain conserved
relaxation or decay has an explicit conserved destination if it moves mass
artifact-only replay reconstructs route use, substrate trace, later routing,
and controls
```
