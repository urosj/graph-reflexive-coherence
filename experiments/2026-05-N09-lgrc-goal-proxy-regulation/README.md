# N09 LGRC Goal-Proxy Regulation

N09 asks whether LGRC can regulate a runtime-visible condition through
serialized proxy evidence, memory-shaped route choice, and artifact-replayable
control loops without treating the result as intention, agency, reward
optimization, or biological behavior.

The core question is:

```text
Can a runtime-visible proxy condition be measured, compared to a declared
target band, and repeatedly steered by route choice or packet scheduling while
preserving identity/support, memory provenance, budget accounting, and clean
claim boundaries?
```

Arc-of-Becoming orienting question:

```text
Can this system regulate something it can observe, using memory-shaped choice,
while preserving identity/support and clean claim boundaries?
```

This is a bounded question, not a claim. It asks what kind of regulation-like
pattern appears under controlled probes and keeps failure modes useful:

```text
no regulation
wrong-direction response
probe-supported correction
repeated bounded correction
support-dependent regulation
identity-anchored regulation
artifact-only regulation candidate
native-policy gap or native expression candidate
```

N09 follows the completed lower foundation:

```text
N05:
    O5 self_sustained_oscillator_candidate
    O6 route coupling blocked by missing_route_conductance_memory_policy

N06:
    SC6 artifact_only_semantic_route_choice_candidate

N07:
    ID6 artifact-only source-specific bounded_non_destructive_exchange
    runtime identity acceptance remains blocked

N08:
    Hypothesis A:
        artifact_only_route_memory_or_trail_affordance_candidate
        scope = artifact_only_serialized_producer_policy_route_memory_or_trail

    Hypothesis B:
        static_positive_geometry_route_response_persistence_candidate
        blocker = native_route_conductance_memory_policy_missing
```

N09 may consume the scoped N08 Hypothesis A memory/trail result and may cite
the Hypothesis B static geometry route-response result as a design direction.
It must not consume native geometry-mediated trail support, pure flux trail
memory, ACO, agency, intention, identity acceptance, locomotion, biological, or
unrestricted movement claims.

## Roadmap Position

The N05-N11 roadmap orders the work as:

```text
producer-mediated evidence
-> runtime-visible triggers
-> native route choice
-> identity invariance
-> memory-shaped choice
-> goal-proxy regulation
-> identity-continuous cross-context integration
```

N09 is the first regulation experiment. Its job is to show whether the prior
layers can be used to keep a measured condition within, toward, or around a
declared proxy target. It separates regulation from intention.

N09 maps to the roadmap's A4 rung:

```text
A4 = goal-proxy regulation of a runtime-visible condition
```

N09 may observe A5-relevant facts, such as whether regulation preserves or
disrupts N07 identity/support evidence, but it does not close A5. A5 belongs
to N10 unless a later N09 extension explicitly opens identity-continuous
regulation as its own branch.

N09 must therefore record identity/support outcomes as handoff tags:

```text
identity_not_tested_under_regulation
identity_preserved_under_regulation
identity_disrupted_under_regulation
support_preserved_under_regulation
support_disrupted_under_regulation
```

## Hypotheses

N09 keeps two hypotheses separate.

```text
Hypothesis A:
    serialized producer/policy goal-proxy regulation

Hypothesis B:
    native substrate-mediated goal-proxy regulation
```

The initial Iterations 1-9 follow Hypothesis A, but Hypothesis B is not
discarded. It is staged:

```text
B0:
    inventory native/substrate-mediated ingredients already available from
    N05-N08 and record whether they can carry proxy/error/response evidence

B1:
    after the A-path identifies load-bearing proxy variables and response
    laws, try a geometry/substrate-mediated regulation probe that does not use
    experiment-local correction logic

B2:
    if a pure native regulation loop requires an unavailable LGRC policy
    surface, record the blocker explicitly rather than converting the A-path
    scaffold into a native claim
```

Expected initial native blocker:

```text
native_goal_proxy_regulation_policy_missing
```

This mirrors the N05-N11 roadmap discipline: use producer scaffolds to discover
mechanism shape, then consider native absorption only after the needed policy
surface is specific.

## Goal-Proxy Definition

For N09, a goal proxy is a serialized artifact-visible regulation target over
a runtime-visible condition.

Required proxy fields:

```text
proxy_id
proxy_kind
regulated_variable_id
regulated_variable_surface
regulated_variable_digest
measurement_value
target_band
error_metric
error_value
event_time_key
scheduler_event_index
proxy_policy_id
proxy_policy_digest
node_plus_packet_budget_before
node_plus_packet_budget_after
node_plus_packet_budget_error
source_artifacts
source_reports
claim_flags
```

Examples of acceptable regulated variables:

```text
source reservoir coherence band
target support-area coherence band
dual-basin exchange balance
route-use frequency balance
oscillator return amplitude band
memory/trail surface threshold band
```

Invalid proxy sources:

```text
hidden reward
fixture label such as food or danger without runtime evidence
experiment-side if/else
report-side correction
post-hoc target changes
unserialized Python state
agent intention or desire labels
```

## Regulation Definition

For N09, regulation means:

```text
measured runtime-visible proxy state
-> serialized error / target-band comparison
-> route choice, producer scheduling, or packet work selected from runtime
   evidence
-> later proxy state moves toward, remains within, or recovers toward the
   declared band
-> artifact replay reconstructs the chain
```

Regulation does not require the system to know the target semantically. It only
requires replayable proxy/error evidence and causal effect through allowed LGRC
boundaries.

The first N09 tranche is single-variable regulation. Multi-variable,
cross-context, or adaptive policy composition is N10 material unless N09 later
adds an explicit extension. N09 may record multi-variable needs as handoff
requirements, but should not blur them into the GPR ceiling.

The default fixture should start with a directly budget-auditable node
coherence band. Oscillator-return regulation is a secondary fixture family for
closing the N05 -> N09 loop after the simple proxy path is understood.

Memory-shaped regulation is a required N09 lane, not just optional wording.
At least one GPR3+ positive path should compare:

```text
same proxy / same target / same policy with N08 memory surface
vs
same proxy / same target / same policy without N08 memory surface
```

This isolates whether the regulation loop actually consumes the N08
memory-shaped-choice layer. The no-memory comparator must either fail with a
specific blocker or produce a different response classification.

## GPR Ladder

The GPR ladder is local to N09. It is an evidence classification, not a claim
flag.

```text
GPR0:
    no regulation. The proxy condition is unmeasured, label-only, or drifts
    without a runtime-visible response.

GPR1:
    proxy measurement. A runtime-visible regulated variable and target band are
    serialized with digest and event order.

GPR2:
    error signal. Proxy error is computed from serialized runtime-visible
    evidence under a declared policy.

GPR3:
    proxy-conditioned action eligibility. A producer or native arbitration path
    changes eligibility, candidate score, or scheduling because of the proxy
    error, without mutating state directly.

GPR4:
    single-cycle correction. The selected route or scheduled packet work
    measurably reduces error or returns the proxy condition toward the target
    band in one replayable cycle.

GPR5:
    repeated bounded regulation. Multiple cycles keep the proxy condition
    bounded, recover from small perturbation, or correct alternating deviations
    under the same serialized policy. If N07 support evidence is consumed,
    identity/support preservation or disruption must be tagged.

GPR6:
    artifact-only goal-proxy regulation candidate. Artifact replay reconstructs
    proxy measurement, error computation, route/memory/producer evidence,
    scheduled or processed packet work, resulting proxy-state changes,
    identity/support outcome tags where applicable, and controls without
    private runtime state.
```

## Native And Producer Scope

N09 is experiment-local unless a separate Phase 8/core task is opened.
Producers may be used as instrumented scaffolds when they:

```text
observe committed runtime-visible state
emit proxy/error/eligibility records
schedule through LGRC queues
leave mutation to step() or committed topology machinery
avoid coherence, packet ledger, topology, and claim writes
```

Every positive result must classify the mechanism status:

```text
producer_mediated
threshold_authorized
native_route_arbitrated
memory_shaped
identity_anchored
constitutive_native
native_policy_gap
```

`memory_shaped` and `identity_anchored` are N09-local extensions to the
roadmap status tags. They describe which upstream evidence was consumed; they
do not by themselves make the mechanism constitutive native.

If current LGRC cannot express the needed proxy, error, threshold, or
regulation surface as serialized native policy, N09 must record a native-policy
gap rather than hiding the missing mechanism in fixture code.

N09 follows the same dissolution discipline as earlier experiments:

```text
producer-mediated regulation scaffold
-> threshold-authorized regulation
-> native route-arbitrated regulation
-> constitutive native policy only if current LGRC can express it
```

The experiment should discover which producer decisions are essential and
which are bookkeeping. Native absorption is deferred until the artifacts show
the minimal policy surface.

## Ceiling Algorithm

The N09 ceiling is the highest GPR level whose required gates pass. If a later
gate fails, the ceiling falls back to the strongest passing level and records a
primary blocker.

Examples:

```text
GPR5 passes but artifact replay fails:
    ceiling = repeated_bounded_proxy_regulation_candidate
    blocker = artifact_only_goal_proxy_replay_failed

GPR4 corrects once but repeated windows fail:
    ceiling = single_cycle_proxy_correction_candidate
    blocker = repeated_regulation_not_bounded

GPR3 changes eligibility but correction is absent or wrong direction:
    ceiling = proxy_conditioned_route_selection_candidate
    blocker = no_response_to_error or wrong_direction_response
```

Identity/support checks are handoff gates. Failure there records
`identity_disrupted_under_regulation` or `support_disrupted_under_regulation`
and blocks N10 consumption, but does not automatically erase lower N09 proxy
regulation evidence if the proxy-regulation chain itself remains valid.
