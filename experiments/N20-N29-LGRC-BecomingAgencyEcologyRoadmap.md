# N20-N29 LGRC Becoming-Agency Ecology Roadmap

This roadmap defines the active experiment arc after the closed N12-N19
agency-prerequisite and native-readiness stack. N20 is now closed as the
translation-contract entry point, and N21 is the next experiment.

It is not a Phase 8 implementation plan, not an ant ecology specification, and
not an agency claim. Its purpose is to ground the deeper "agency of becoming"
question in LGRC-visible producer artifacts before any applied ecology work is
specified.

## Starting Boundary

N19 closed the AP3-AP8 native-readiness review with a specific blocker:

```text
full_ap3_ap8_nat4_ladder_generation_supported = false
current_implementation_can_generate_claimed_ap_ladder = false
claimed_ladder_generation_status = blocked_by_ap4_ap5_nat4_evidence_gaps
```

Therefore N20-N29 must not assume that the AP3-AP8 stack is natively
generatable. AP4/N14 and AP5/N15 remain the key native-readiness gaps, and
Phase 8 remains unopened.

The next question is not:

```text
Can we claim agency now?
```

It is:

```text
Which becoming-style agency primitives can be expressed in LGRC-visible
geometry, and which still require explicit producer surfaces?
```

## Three Converging Paths

N20-N29 should keep three paths distinct until they converge in N29.

### 1. Geometric Coherence Papers And Essays

The geometric coherence path supplies the interpretive method and diagnostic
language. The Arc of Becoming sequence supplies the method:

```text
Classification of Becoming
Interrogation of Becoming
Naturalization of Becoming
Cultivation of Becoming
```

The essays supply the agency-specific interpretation needed to use that method
without reducing agency to choice, labels, or performance:

```text
From Structural Abundance to Agency
Agency of Becoming: An Interpretation Through Reflexive Coherence
Agency After Choice
Sentience as Read-Back
```

Reading boundary:

```text
From Structural Abundance to Agency:
    abundance as basin proliferation and surplus possibility, but not yet
    agency.

Agency of Becoming:
    agency as persistence capacity across perturbation, withdrawal, transfer,
    and proxy pressure.

Agency After Choice:
    broader vocabulary for scaffolded, native, proxy, wounded, extractive, and
    generative agency.

Sentience as Read-Back:
    boundary source separating agency from sentience; read-back/interiority
    should remain blocked unless explicitly tested in a later series.
```

These are method and interpretation sources. They do not by themselves prove
LGRC agency, native support, sentience, biological behavior, semantic
intention, or identity acceptance.

### 2. Graph Substrate Experiments

The graph substrate path is this repository's executable and artifact-backed
record. N01-N19 provide:

```text
graph primitives
LGRC producer artifacts
AP3-AP8 artifact-level prerequisites
NAT classification discipline
claim-boundary controls
replay and digest practice
```

N20-N28 should remain here and test becoming primitives close to existing LGRC
implementation surfaces.

### 3. Agentic Ecology / Ants Project

The agentic ecology path supplies the applied domain: multi-basin ecology,
shared-medium traces, ant/colony-like support flow, producer residue, and
medium debt.

Before N29, this path is context only. Do not write formal ants ecology specs
or claim applied ant behavior inside N20-N28.

## Arc of Becoming Method

N20-N29 should use the Arc of Becoming as method:

```text
Classification:
    name the LGRC-visible primitive without semantic promotion.

Interrogation:
    probe the boundary where the primitive fails, mutates, or becomes
    distinguishable from a proxy.

Naturalization:
    record what is carried by source-current geometry and what remains explicit
    producer scaffolding.

Cultivation:
    add the minimal producer surface needed to test the primitive without
    relabeling scaffolded success as native support.
```

The agency-of-becoming diagnostics become experiment design constraints:

```text
withdrawal resistance
naturalization depth
substrate or configuration transfer
proxy divergence / proxy collapse pressure
generative versus extractive persistence
```

Use geometric wording where possible:

```text
basin signature
support/coherence floor
boundary integrity
flux balance
live continuation set
susceptibility update
surplus-supported optionality
proxy-geometry divergence
basin-forming capacity
```

Avoid treating "learning", "choice", "function", "intention", "goal", or
"agent" as primitive labels.

A producer is an explicit implementation surface that introduces, updates,
routes, schedules, labels, or preserves a quantity that is not yet carried by
LGRC source-current geometry. A producer is not the agent; it is the current
implementation residue left after attempting to express apparent agent state as
basin, boundary, support, flux, coupling, aftereffect, susceptibility, trace,
reserve, or persistence condition.

Every producer variable should be classified as:

```text
substrate_carried
producer_mediated
naturalization_debt
blocked_relabel
```

N20 defines the admissibility contract for the rest of the arc. It does not
produce withdrawal resistance, naturalization depth, learning, choice, spark,
abundance, ecology, or agency evidence.

## N22. Susceptibility Update / Durable Geometry Modification

Question:

```text
Can prior interaction durably alter later basin geometry without becoming a
semantic learning label?
```

LGRC reading:

```text
learning = durable geometry modification visible in later replay or transfer
```

Artifact labels should prefer:

```text
susceptibility update
aftereffect
durable geometry modification
```

`Learning` is the bounded interpretation, not the primitive label.

The first easy case is reinforcement:

```text
prior route use -> altered future route susceptibility
```

But learning is broader than reinforcement. N22 should classify learning modes
without claiming all of them at once:

```text
L0 = no durable geometry change
L1 = reinforcement / susceptibility shift
L2 = inhibitory learning / reduced susceptibility
L3 = boundary reshaping
L4 = corridor formation
L5 = basin splitting / sub-basin differentiation
L6 = new basin formation candidate, deferred to N25
```

N22 should contribute to the AP4/AP5 gap story by producing source-current
susceptibility evidence, but it should not try to prove choice/collapse or
new-basin formation.

Blocked claims:

```text
intention
semantic choice
semantic action
agency
free will
```

## N23. Live-Continuation Collapse / Selection Geometry

Question:

```text
Can multiple live continuation branches resolve into one selected continuation
without semantic choice labels or producer-side selection?
```

LGRC reading:

```text
choice = collapse among multiple live continuation branches with real
         counterfactual alternatives
```

In generated artifacts, the primary label should be `live-continuation
collapse` or `selection geometry`. `Choice` remains interpretive shorthand
only.

N23 should directly address the AP4/N14 NAT4 gap by making route or branch
selection source-current and geometry-conditioned, not merely a constructed
followout, declared preference, or label.

Expected controls:

```text
fake alternatives blocked
single-branch relabel blocked
post-hoc selected branch blocked
producer preference injection blocked
random tie masquerading as collapse blocked
semantic choice / intention relabel blocked
```

N23 should not claim agency, free will, semantic choice, or intention. It is a
collapse/selection geometry experiment.

## N24. Abundance As Surplus-Supported Optionality

Question:

```text
Can surplus support above the maintenance floor open optional continuations
without collapsing basin integrity?
```

LGRC reading:

```text
abundance = support/coherence surplus that permits optionality, exploration,
repair, specialization, or splitting without dropping below maintenance floors
```

N24 should distinguish abundance from reward maximization or proxy gain. A
positive row should show optional continuation space created by surplus while
negative controls reject hidden budget relief, floor crossing, and proxy-only
improvement.

## N25. Spark / Sub-Basin / New-Basin Formation

Question:

```text
Can becoming pressure produce a new distinguishable basin or sub-basin, rather
than only reinforcing an existing basin?
```

LGRC reading:

```text
spark / new-basin formation = a source-backed bifurcation or basin-creation
candidate with distinguishable boundary, support, and replay state
```

N25 should be separate from learning because creating a new basin is not merely
reinforcement. It should depend on declared surplus/support conditions from
N24 or record an explicit blocker when those conditions are absent. It should
distinguish:

```text
reinforced old basin
reshaped old boundary
sub-basin differentiation
new basin formation candidate
transient fluctuation misread as basin
```

Expected controls:

```text
label-only new basin rejected
single-basin thickening relabel rejected
merge/leakage masquerading as new basin rejected
non-replayable transient rejected
hidden producer insertion rejected
```

## N26. Proxy Divergence / Proxy Collapse

Question:

```text
Can proxy improvement diverge from basin deepening?
```

LGRC readings:

```text
proxy divergence = proxy metric improves while basin persistence capacity
                   stalls or degrades
proxy collapse = proxy-optimized success fails under a perturbation that
                 basin-deepened success would survive
```

N26 should address the AP5/N15 NAT4 gap by making proxy derivation and
lower-stack inputs source-current under replay, with explicit controls against
post-hoc target formation, hidden proxy policy, and proxy-only improvement.

## N27. Configuration / Substrate Transfer

Question:

```text
Can a basin signature transfer across declared substrate or configuration
changes without reconstructing the original support?
```

LGRC reading:

```text
transfer = basin signature persists under a structurally distinct fixture,
           context, or substrate mapping
```

N27 should keep transfer separate from proxy divergence. A transfer row should
declare what changes, what remains the same basin signature, which supports are
reconstructed, and whether any reconstruction becomes producer residue or
naturalization debt.

Near-term transfer should be configuration-first:

```text
configuration / fixture / topology transfer
```

Substrate transfer is allowed only when the substrate mapping is explicitly
declared and source-backed.

## N28. Generative Vs Extractive Persistence

Question:

```text
Can one basin persist while increasing the basin-forming capacity of its
environment or neighborhood?
```

LGRC reading:

```text
generative persistence = focal basin remains stable while neighboring or
sub-basin structures become more distinguishable, better supported, or more
capable of preserving their own boundaries
```

N28 should measure environment-side effects, not only focal basin survival:

```text
neighbor_basin_count_or_distinguishability
neighbor_support_floor
sub_basin_survival
environment_basin_forming_capacity
focal_basin_extraction_cost
extractive_flattening_score
```

This is the first experiment in the arc that should touch ecology-like
structure, but still inside this repository and still without ant-specific
specification. It should distinguish:

```text
generative persistence
competitive persistence
extractive persistence
merge/leakage masquerading as support
```
