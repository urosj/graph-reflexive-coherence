# Hypothesis B: Support-Seeking Regulation

Hypothesis B states that support-seeking regulation can be distinguished from
external proxy regulation when the target condition is derived from
source-current support state, serialized, replayable, budgeted, and
control-clean.

## Question

```text
Can a bounded regulation response preserve or restore a support-state-derived
condition in a way that is not reducible to an externally declared proxy, hidden
producer target, post-hoc support label, or budget-ambiguous correction?
```

## Required Surfaces

```text
source-current support-state fields
support-condition derivation rule
runtime-visible support target
support error signal
bounded response magnitude
node-plus-packet budget debit
artifact replay record
support trend/stability fields
```

## Required Controls

```text
external proxy only -> blocked
hidden support target -> blocked
post-hoc support label -> blocked
support-disrupted regulation -> blocked
stale source replay -> blocked
budget ambiguity -> blocked
identity acceptance relabel -> blocked
semantic goal ownership relabel -> blocked
agency relabel -> blocked
```

## Expected Conservative Result

```text
AP3 self-maintenance-candidate evidence may be supported if controls pass
external proxy regulation remains separated
support-derived target remains non-semantic
identity acceptance and agency remain blocked
native support remains false unless separate Phase 8 implementation exists
```
