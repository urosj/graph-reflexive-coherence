# Hypothesis C - Claim Boundary And Producer / Native Discipline

## Claim

N25.2 can preserve the distinction between runtime evidence, producer-mediated
scheduling, native substrate claims, and downstream N26 consumption.

## Expected Support

Hypothesis C is supported only if every N25.2 artifact records:

```text
producer scheduling does not become native support
producer compatibility audit remains load-bearing
runtime mutation ownership remains with LGRC9V3 transitions
MB5 candidate evidence does not become MB6 by relabel
N25 / N25.1 requirements do not become runtime evidence
Phase 8 examples do not become proof by visualization
runtime probes use the closed implementation and record source-tree digests
implementation defects are recorded only as blockers or repair targets
N26 unscoped consumption remains false unless MB6 passes
unsafe claim flags remain false
```

## Failure Conditions

Hypothesis C fails if N25.2:

```text
uses labels instead of source-backed evidence
uses producer-scaffold success as native support
promotes MB5 to MB6 without a gate
converts N25 BF5 scoped sub-basin evidence into independent multi-basin evidence
patches source, spec, test, example, or implementation files to make evidence pass
uses replay/reconstruction as a replacement for original runtime execution
claims semantic learning, choice, agency, sentience, ant ecology, organism/life,
Phase 8 completion, or unrestricted autonomy
```

## Cross-Hypothesis Rule

Hypotheses A or B may pass only if Hypothesis C also passes. If Hypothesis C
fails, positive MB5 or MB6 classifications are demoted or blocked regardless of
their local margins.
