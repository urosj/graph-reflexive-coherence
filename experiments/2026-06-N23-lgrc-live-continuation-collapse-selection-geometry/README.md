# N23 - LGRC Live-Continuation Collapse And Selection Geometry

N23 is a closed becoming-primitive experiment after N22. It consumes the N20
contract row for `live_continuation_collapse` and the N22 closeout only as
bounded producer-mediated susceptibility-update context, then tests whether
multiple live continuation branches can resolve into one selected continuation
without semantic choice labels or producer-side selection.

Core question:

```text
Can multiple source-current live continuation branches collapse into one
continuation while the alternatives remain auditable as real counterfactual
branches, not fake labels, producer preferences, random ties, or post-hoc
selected-branch stories?
```

N23 is not a semantic choice, intention, free will, or agency experiment.
`Choice` may be used only as bounded interpretation for collapse among real
live continuation alternatives. The primitive labels are:

```text
live_continuation_collapse
selection_geometry
```

## Primitive Reading

Live-continuation collapse means:

```text
at least two live, boundary-distinguishable continuation branches exist before
collapse; one continuation becomes the source-current continuation; the
non-selected branches remain auditable as real counterfactual alternatives
rather than disappearing into labels or post-hoc reconstruction.
```

For N23, a live branch counts only when it is recorded:

```text
in the same source-current run
in the same declared branch window
before the collapse-window start
with branch-specific support/coherence traces
with branch-specific boundary/flux traces
```

Branches assembled across independent runs, replay forks, report-side
construction, or post-hoc alternative labels do not count as the original live
branch set. Replay forks may audit counterfactuals, but they cannot create the
pre-collapse live-branch evidence.

Counterfactual branch retention means:

```text
the non-selected branches remain as immutable source-current pre-collapse
audit records.
```

It does not require the non-selected branches to remain dynamically active
after collapse, and it is not satisfied by labels, producer alternatives,
replay-created branches, or report-side reconstruction.

Selection geometry means:

```text
the selected continuation is explained by source-current support, coherence,
boundary, flux, susceptibility, or route-conditioned geometry, not by a
producer preference label or semantic choice claim.
```

The selected-branch reason must come from traces available before or during the
collapse window. It cannot be assigned after report construction.

N23 should directly address the AP4/N14 NAT4 gap by making route or branch
selection source-current and geometry-conditioned. AP5 remains conditional:
only rows that use proxy derivation or target formation may carry AP5
dependency.
